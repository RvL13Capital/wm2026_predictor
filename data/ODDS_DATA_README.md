# Supplying the Gate-G2 Odds Data (`wc{2014,2018,2022}_odds.csv`)

Gate G2 (`backtest_real_market.py`) decides whether the edge scanner may ever
leave paper mode. It needs **real historical closing 1X2 odds** for the three
World Cups. The CSVs in this directory are **fixture-prefilled templates**
(generated by `scripts/make_odds_templates.py`): teams, orientation, and phase
are already correct — *you only type three numbers per row.*

## What to enter

For each match, the pre-kickoff **closing** odds of the standard **1X2 / Match
Odds** market (90 minutes, draw possible — for KO games too):

| Column | Meaning |
|---|---|
| `odds_home` | Decimal odds on `team_a` winning in 90' (e.g. `1.85`) |
| `odds_draw` | Decimal odds on the 90' draw (e.g. `3.40`) |
| `odds_away` | Decimal odds on `team_b` winning in 90' (e.g. `4.50`) |
| `bookmaker` | Where the price came from (`pinnacle_close`, `oddsportal_avg`, `bet365`, …) |
| `source_url` | Optional provenance link |

Rules of the road:

- **Decimal (European) odds**, each strictly > 1.0.
- **1X2, not "to qualify" / "to advance"** — KO matches must have a draw price.
  (The harness already settles ET games correctly: a KO game that went to
  extra time counts as a 90' draw.)
- **Closing odds** (last pre-kickoff), or the closest you can get — note what
  it is in `bookmaker`. Prefer Pinnacle close; an odds-comparison "average
  close" is acceptable; opening odds only as a last resort (label them!).
- Don't guess and never interpolate — **leave the row blank** if you can't
  source it. Blank rows are skipped with a count; partial files run fine.
- If `team_a/team_b` appear swapped at your source, just swap the home/away
  odds you type — or don't: the loader auto-detects orientation against the
  results file. German spellings also resolve automatically.

## Where to get it

1. **Oddsportal archive** (free): oddsportal.com → Football → World Cup →
   season 2014/2018/2022 → Results. Each match page lists average and
   per-book closing 1X2. ~64 matches per tournament ≈ 1–2 h of manual entry.
2. **Kaggle**: search "world cup 2018 odds" / "world cup 2022 odds" — several
   datasets carry bookmaker 1X2; spot-check ~5 matches against Oddsportal
   before trusting one.
3. **Betfair historical data** (account needed): exchange closes, fine if
   labelled `betfair_close`.
4. *Not* football-data.co.uk — it covers club leagues only, no World Cups.

**Priority order: 2022 → 2018 → 2014.** The verdict's criterion (a) needs the
model to beat the close on ≥ 2 folds, so 2018 + 2022 is the minimum viable
gate; 2014 strengthens it.

## Workflow

```bash
# 1. (already done) templates exist:
python3 scripts/make_odds_templates.py            # idempotent; never clobbers filled rows

# 2. Fill a few 2022 rows, then sanity-check the parsing early:
python3 backtest_real_market.py --years 2022      # partial fills are fine

# 3. Finish 2022 + 2018 (and ideally 2014), then run the gate:
python3 backtest_real_market.py                   # writes validation/backtest_real_market.txt

# 4. Commit the filled CSVs + the verdict output (provenance!).
```

The pre-registered verdict (do not reinterpret after seeing results): real-money
use only if model/blend log-loss beats the de-vigged close on **≥ 2 of 3
tournaments** AND the flat-stake ROI 95% CI **excludes 0**. Anything else →
the scanner stays paper-only, and that outcome is recorded as a success
(loss prevention).
